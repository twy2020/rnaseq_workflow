# 开发边界说明

## 1. old_prj 的定位

`old_prj` 是历史原型目录，只用于理解项目最初想解决的问题：

- 批量处理 RNA-seq 数据。
- 从 SRA/FASTQ 到质控、修剪、比对、定量。
- 输出表达量结果。
- 提供日志、进度和基础交互。

`old_prj` 不作为新系统的代码基线。

## 2. 明确不做的事

后续开发中不做以下事情：

1. 不在 `old_prj` 内修复 bug。
2. 不把老项目文件直接复制为新模块实现。
3. 不以老项目的目录结构作为新系统结构。
4. 不兼容老项目中不稳定或错误的配置命名。
5. 不把老项目的流程缺陷带入新系统。

## 3. 后续开发位置

新系统核心代码位置：

- `workflow/rnaseq_workflow`

模块设计说明位置：

- `modules`
- `docs`

配置示例位置：

- `config`

运行输出位置：

- `runtime_logs`
- 后续可配置到项目指定输出目录

## 4. 老项目如何被使用

老项目只用于三类参考：

1. 提取需求：确认需要哪些功能模块。
2. 识别风险：发现旧实现中哪些问题应在新系统中规避。
3. 辅助论文：作为“历史原型与需求来源”的材料。

所有新功能都应按新架构重新设计和实现。
